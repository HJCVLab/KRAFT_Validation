# KRAFT Validation and Reuse Code
[![DOI](https://zenodo.org/badge/DOI/10.5281/zenodo.21320079.svg)](https://doi.org/10.5281/zenodo.21320079)

This repository contains the reproducible Python scripts used for the Technical Validation section of the KRAFT manuscript, together with examples for linking apartment transaction records to monthly school-count and annual private-education indicators.

## Repository scope

The files in this repository reproduce the validation statistics, tables, figures, metadata checks, and education-indicator merge examples. If the manuscript states that full preprocessing code is publicly available, the source-specific preprocessing scripts used to construct the released CSV files must also be added before submission.

## Recommended directory layout

The commands below assume that the extracted dataset directory is stored next to the code repository.

```text
workspace/
├── KRAFT/                      # this repository
│   ├── KRAFT_run_all_validations.py
│   ├── KRAFT_technical_validation_*.py
│   ├── KRAFT_example_education_join.py
│   ├── KRAFT_example_private_education_join.py
│   ├── kraft_package.py
│   ├── kraft_region_join.py
│   ├── validation_outputs/
│   └── requirements.txt
└── Dataset/
    └── KRAFT_v1.0/
        ├── Data/
        ├── README.md
        └── ...
```

## Files

| File | Purpose |
|---|---|
| `KRAFT_technical_validation_transaction_records.py` | Computes transaction-level descriptive statistics and the Section 4.2 draft. |
| `KRAFT_technical_validation_auxiliary_indicators.py` | Computes auxiliary-indicator descriptive statistics and the Section 4.3 draft. |
| `KRAFT_technical_validation_distributional_regional_coverage.py` | Computes annual and regional coverage checks and generates Figures 2–4. |
| `KRAFT_technical_validation_metadata_consistency.py` | Checks documentation, metadata coverage, region mapping, and validation-report status. |
| `KRAFT_run_all_validations.py` | Runs all four validation scripts sequentially. |
| `KRAFT_example_education_join.py` | Demonstrates a monthly transaction-to-school-count merge using a parent-city Sigungu join key. |
| `KRAFT_example_private_education_join.py` | Demonstrates an annual private-education merge using calendar year and a parent-city Sigungu join key. |
| `kraft_package.py` | Reads either a ZIP package or an extracted release directory through the same logical paths. |
| `kraft_region_join.py` | Provides the reusable Sigungu join-key function. |

## Supported input layouts

The scripts accept either a ZIP file or an extracted directory. For extracted data, `--input` may point to the release root containing `Data/` or to its parent directory. ZIP files with or without a single top-level `KRAFT_v1.0/` directory are also supported.

## Installation

Python 3.10 or later is recommended.

```bash
python -m venv .venv
```

Windows PowerShell:

```powershell
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
pip install -r requirements.txt
```

Linux or macOS:

```bash
source .venv/bin/activate
python -m pip install --upgrade pip
pip install -r requirements.txt
```

## Run all validations with the extracted dataset

Windows PowerShell:

```powershell
python KRAFT_run_all_validations.py `
  --input "..\Dataset\KRAFT_v1.0" `
  --output-dir "validation_outputs"
```

Linux or macOS:

```bash
python KRAFT_run_all_validations.py \
  --input ../Dataset/KRAFT_v1.0 \
  --output-dir validation_outputs
```

A ZIP archive can be used instead by replacing the input path with the archive path.

## Generated and archived outputs

The `validation_outputs/` directory contains the 19 files generated by `KRAFT_run_all_validations.py`, including:

- Table 4 and Table 5 statistics
- full auxiliary descriptive statistics
- annual, Sido, and Sido-month count tables
- the price-per-square-meter distribution summary
- Figures 2–4
- metadata consistency reports
- reproducible Section 4 markdown drafts

These archived outputs make the manuscript statements that the generated statistics, figures, and count tables are provided with the validation code accurate.

## Monthly school-count join example

Transaction files can contain general-gu labels such as `용인시 수지구`, whereas the monthly education file is aggregated at the parent-city level `용인시`. The script creates `Education_Join_Sigungu` without overwriting the original `Sigungu`.

```bash
python KRAFT_example_education_join.py \
  --input ../Dataset/KRAFT_v1.0 \
  --year 2024 \
  --sample-output examples/KRAFT_Education_Join_Sample_2024.csv
```

## Annual private-education join example

Annual private-education observations use year-end `YearMonth` keys. The example joins by calendar year, not by treating `YYYY12` as a December-only observation.

```bash
python KRAFT_example_private_education_join.py \
  --input ../Dataset/KRAFT_v1.0 \
  --year 2024 \
  --sample-output examples/KRAFT_Private_Education_Join_Sample_2024.csv
```

## Reusable parent-city operation

```python
from kraft_region_join import add_education_join_keys

transactions = add_education_join_keys(transactions)
education = add_education_join_keys(education)
```

Only labels matching the form `<city> <general-gu>` are collapsed. Standalone metropolitan districts such as `강남구` remain unchanged.

## Notes

The scripts do not modify the original dataset. All validation results are written to the directory specified by `--output-dir`.
